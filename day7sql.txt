Constraints:
-----------
constraints are rules applied to a table columns in MySQL to ensure that only valid and accurate data is stored in the database.

why to use constraints?
----------------------
-> to maintain data integrity
->to prevent invalid data entry
->avoid duplicate records
->ensures relations between tables 
->makes the database reliable and constrained.

Types of constraints:
--------------------
1.not null constraint 
:doesn't allow null values(allows duplicates)

2.unique key 
: it does not allow duplicate values(duplictae null values is allowed)

3.primary key (column  and table level)
:it is the combination of unique+not null

4.composite key
:apply primary key at two or more colums
:primary key at table level (considers as key:value pairs consider all the columns mentioned in it as a single record)

5.foreign key(***)
: a foreign key is a key used to link two tables together.

->a foreign key is afield or collection of fields in one table that refers to the primary key in another table.

->the table containing the foreign key is called child table and the table containing the candidate key is called references table or parent table.


6.check constraint
: checks condition if true only the command works

7.default constraint
:to take systems date and time defaultly 


REFERENTIAL ACTIONS:
-------------------
referential actions define what happens to the child table data when parent table data is updated or deleted.

they are used with foreign key constraints.

1.on delete cascade
when a parent table row is deleted all matching child rows are automatically deleted

2.on update cascade
when a parent key is updated all matching child foreign keys are updated automatically
